What's New in Fleet Manager

Latest updates are always shown at the top (newest first).
This page shows recent changes in the software so users can quickly see what is new or improved.
Latest Backup, Drivers (search/print/export/post), Import (list-only + progress)
Newest changes
  • Backup: Download and Send to Email only (Save to Path removed). Scheduled backup runs daily and sends backup to email only. Left menu: Backup link above What's New.
  • What's New: Always shown at the bottom of the left menu (below Backup).
  • Import only on list pages: Import Excel button removed from Driver, Vehicle, and Parking entry forms. Import is available only from Drivers List, Vehicles List, and Parking List.
  • Import progress: When uploading a file for import, an overlay shows file name, upload percentage, and “Processing…” so users know the system is working and don’t refresh.
  • Driver delete: Deleting a driver from the list now correctly removes related records (status changes, transfers, attendance) so no database error occurs.
  • Driver list search: Search filters only by columns shown on the list (Driver ID, Name, CNIC No, CNIC Status, License No, License Status, Phone, Driver District). You can search by “Valid” or “Expired” for status.
  • Driver print/PDF & Export Excel: Print view and Excel export use the same columns as the list (Driver ID, Name, CNIC No, CNIC Status, License No, License Status, Phone, Driver District). Export now downloads a real .xlsx file.
  • Driver form – Post: Post dropdown shows only full name (short name removed). On edit, if the driver’s post was set by import and is not in the master list, it is kept and not changed.
  • Driver import: CNIC Status and License Status are set automatically from expiry dates during import (Valid/Expired). License No “0” or blank is treated as invalid and the row is skipped with an error.
  • Driver list delete: CSRF token added to the delete form so “Bad Request” no longer appears when deleting a driver.
Excel import templates, strong validation & error tables
Previous update
  • Excel templates with highlighted required columns: Parking, Vehicles, and Drivers import templates are now real .xlsx files. All required columns have colored + bold headings so users can immediately see which fields must be filled.
  • Per‑row validation on import: During import, every row is checked for required fields and data type (e.g. capacity, driver capacity, dates). Duplicate checks follow the same rules as forms: Parking Station name + district, unique Vehicle No, unique Driver ID, CNIC, and License No.
  • Safe “all‑or‑nothing” import: If even one row has an issue, the entire file is rolled back and no partial data is saved, keeping your database clean.
  • Detailed error table on screen: Import pages now show a column‑wise error table (Row #, key column, Issue) that stays visible until you fix and re‑upload, so the user can easily match Excel rows with problems.
  • Cleaner text values on import: Optional text fields that are left blank in Excel are stored as N/A instead of technical NaN values.
  • Login username made case‑insensitive: Admin, admin, or ADMIN all work with the same password, making login more user‑friendly.
  • Party & Product lists pagination/search: Party and Product Name lists now use the same professional pagination and backend search as other lists (records per page, page numbers, search across all pages).
Login screen, employee module enhancements & clean reports
Most recent updates
  • Login screen added: Simple login form with default credentials admin / admin to control access (can be extended later for real users).
  • Employee Posts (Master): New master for all employee posts (short name + full name + remarks) used in Driver and Employee forms.
  • Employees Management: New Employees list and entry form to register non-driver staff (Accounts Manager, Project Manager, etc.) with strong validation and history-based department suggestions.
  • Employee details: Employee form now includes Father Name, Place of Birth, DOB, Education, Marital Status, District, Address, CNIC, and bank information, with most fields required.
  • Employee CNIC & reports: CNIC No added to Employees list, CSV export, and print/PDF report view.
  • Driver Form – Post field: Post dropdown is now fully driven by Employee Posts master instead of hard-coded values.
  • Parking Station Form validation: Capacity, Create Date, District and Tehsil are now mandatory with clear on-form error messages.
  • Vehicle Entry validation: Required fields strengthened (Vehicle No, Model, Vehicle Type, Engine No, Chassis No, Driver Capacity, Active Date) with visible validation messages.
  • Duplicate checks: User-friendly messages when saving duplicate Vehicle Number or Parking Station Name, instead of raw database errors.
  • Print views cleaned up: All list print pages (Districts, Parking, Vehicles, Drivers, Employees) now open in a clean report-only layout using a dedicated print base, without sidebar or navbar.
